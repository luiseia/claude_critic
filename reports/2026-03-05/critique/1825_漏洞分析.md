## Part 2: 2D AABB 导致旋转框面积过估 — 已确认

### 代码路径

1. `__call__` line 495-496: 将 8 个 3D 角点投影到 2D 图像空间
2. line 502-503: 取投影点的 `min_u, max_u, min_v, max_v` — **这是 2D AABB**
3. line 506: `_compute_valid_grid_ids(min_u, max_u, min_v, max_v, ...)` — 用 AABB 分配 cell

### 面积过估定量分析

对于尺寸 (L, W)、旋转角度 θ 的投影框:
- 真实面积: L × W
- AABB 面积: (L|cosθ| + W|sinθ|) × (L|sinθ| + W|cosθ|)

| 车型 | (L, W) | θ=0° 过估 | θ=30° 过估 | θ=45° 过估 |
|------|--------|-----------|-----------|-----------|
| Car | (4.5, 1.8) | 1.00× | 1.69× | **2.44×** |
| Truck | (12.0, 2.5) | 1.00× | 2.22× | **3.51×** |
| Bus | (11.0, 3.0) | 1.00× | 2.03× | **2.96×** |
| Trailer | (15.0, 2.5) | 1.00× | 2.43× | **4.09×** |

**Truck/Trailer 在非零旋转角下的 AABB 过估是 Car 的 1.3-1.7 倍。**

### 对 Precision 的杀伤机制

1. **膨胀的 cell 覆盖 → 更多正样本**: 一辆 45° 旋转的 Truck，真实应覆盖 ~4 cells，但 AABB 导致覆盖 ~14 cells
2. **边缘 cell 的回归难度极高**: AABB 外延区域内的 cell 需要预测一个远离自身的物体中心 → 回归目标与 cell 位置不一致 → 预测准确率低
3. **IBW 权重进一步被稀释**: 14 cells 的 IBW = 1/14 ≈ 0.071，而真实应为 1/4 = 0.25 → 梯度信号被削弱 3.5×
4. **评估时的 FP 放大**: 模型在 AABB 外延 cell 上可能预测错误的回归值 → 产生额外 FP → 拉低 precision

### 不是所有 cell 覆盖都是等价的

当前代码 (line 459): 所有覆盖 cell 获得**完全相同**的回归目标:
```python
targets_tokens[cid, base_idx : base_idx + 10] = torch.tensor(slot_tokens, device=device)
```

其中 slot_tokens 包含 [marker, cls, gx, gy, dx, dy, w, h, θg, θf] — 这是物体在 BEV 空间的**绝对坐标**。

但一辆 Truck 覆盖 14 个 cell 时:
- 中心 cell: 回归目标指向自身附近 → 合理
- 边缘 cell: 回归目标指向 cell 外的位置 → 模型需要学习"忽略自身位置，预测远处的中心" → 这是一个非直觉的学习任务

**这就是 Precision 无法提升的隐藏原因**: 边缘 cell 的预测本质上是噪声，但它们在评估中被等权对待。

## Part 3: center/around 标记的架构评估

### CEO 提问: 增加 CENTER/AROUND 标记是否导致回归歧义？

**我的评估: 不仅不会导致歧义，反而是解决 Part 2 问题的正确方向。**

当前 marker 有 4 个值: NEAR, MID, FAR, END
- NEAR/MID/FAR 标记距离等级 (slot 0=最近, slot 1=中间, slot 2=最远)
- 没有区分"这个 cell 是物体中心"还是"这个 cell 在物体边缘"

### CENTER/AROUND 方案分析

**方案**: 将 NEAR/MID/FAR 拆分为 NEAR_CENTER/NEAR_AROUND/MID_CENTER/MID_AROUND/FAR_CENTER/FAR_AROUND + END = 7 个 marker

**优点**:
1. CENTER cell: 完整学习 marker + cls + 8 个回归 token (与当前一致)
2. AROUND cell: 只学习 marker + cls (回归 token 的 weight=0)
3. 消除了边缘 cell 的噪声回归目标 → 减少无效梯度
4. 评估时: 只从 CENTER 预测中提取检测结果 → FP 大幅降低

**缺点/风险**:
1. **Marker 词表从 4 → 7**: 增加了 marker 分类难度，但区分 CENTER vs AROUND 比区分 NEAR vs MID 更有视觉特征支撑 (CENTER cell 的 features 更具物体特征)
2. **CENTER 定义模糊**: BEV 中心投影到图像后可能不在任何 cell 中心 → 需要定义"最近的 cell"为 CENTER
3. **AROUND cell 仍然需要 cls**: 否则 marker 正确但 cls 错误的 AROUND cell 会通过 teacher forcing 污染下游

**判决**: CENTER/AROUND 区分是一个正向改动，不会导致回归歧义，反而能消除当前的回归噪声。但实现复杂度中等，需要修改:
- `generate_occ_flow_labels.py`: 增加 CENTER 判定逻辑
- `git_occ_head.py`: 修改 marker 词表 + 回归权重逻辑
- 评估代码: 只评估 CENTER 预测

## Part 4: 深度权重调制的可行性分析

### CEO 思路: 利用相机内参进行"深度权重调制"

当前的训练权重只考虑 IBW (覆盖 cell 数的倒数)，不考虑物体深度。但:
- **近处物体**: 更多像素 → 更丰富的 features → 更容易检测 → 不需要高权重
- **远处物体**: 更少像素 → 更稀疏的 features → 更难检测 → 需要更高权重

### 可行性评估

**方法 1: 基于 Z 值的线性调制**
```python
depth_weight = depth / max_depth  # [0, 1], 远处权重高
final_weight = ibw_weight * (0.5 + depth_weight)  # [0.5, 1.5] 范围
```
- 实现: 仅需修改 `_get_targets_single_based_on_bev` (line 497)
- 风险: 低。depth 信息已在 `gt_projection_info` 中 (line 487)
- 效果: 给远处小物体更多梯度，给近处大物体（已经因 IBW 被压低）进一步降低

**方法 2: 基于投影面积的归一化**
```python
projected_area = (max_u - min_u) * (max_v - min_v)
area_weight = 1.0 / max(projected_area, min_area)  # 面积越大权重越低
```
- 这本质上是另一种形式的 IBW，但在 2D 像素空间而非 cell 空间
- 与 IBW 叠加可能过度压低大物体

**方法 3: 基于 feature 分辨率的权重**
```python
# 物体在 ViT feature map 上覆盖的 patch 数
n_patches = projected_area / (patch_size ** 2)
feat_weight = 1.0 / max(n_patches, 1)  # patch 覆盖越少权重越高
```
- 最物理化的方法：直接反映 feature 表征的丰富程度
- 实现稍复杂，需要 intrinsic 和 patch_size

**推荐: 方法 1 (最简单) + CENTER/AROUND 标记 (解决根本问题) 组合使用。**

但警告: 在 323 张图的 overfitting 场景下，深度调制可能不会带来显著改善。模型有足够参数记住所有物体，问题在于它记不住——这是因为梯度竞争和信号噪声，不是因为权重分配。

## Part 5: 投影公式的二次确认

### BUG-5 补充分析

Line 494-496 投影公式:
```python
safe_Z = np.where(corners_cam[:, 2] < 1e-3, 1e-3, corners_cam[:, 2])
u_all = (K[0, 0] * corners_cam[:, 0] + K[0, 2] * corners_cam[:, 2]) / safe_Z
v_all = (K[1, 1] * corners_cam[:, 1] + K[1, 2] * corners_cam[:, 2]) / safe_Z
```

数学展开:
- u = fx * X/Z + cx (标准针孔模型) ✓
- 当 Z < 1e-3 时: u = fx * X/1e-3 + cx * Z/1e-3

**新发现**: 当 corners_cam[:, 2] < 0 (角点在相机后方):
- `safe_Z = 1e-3` (clamp to small positive)
- 但 `corners_cam[:, 0]` 可能很大 → u = fx * X / 1e-3 → **极大的 u 值**
- 这会产生一个极端外扩的 AABB → 覆盖几乎整个图像

虽然 line 501 `valid_indices = valid_z` 过滤掉了 Z < 0 的角点的 min/max 贡献，但 safe_Z 的 clamp 对 0 < Z < 1e-3 的角点仍然会产生极端投影。这是 BEV 范围边缘(-50m 到 +50m)的物体可能出现的情况。

### 核心结论

**2D AABB 的面积过估是 Precision 无法提升的又一根因。** Truck 在 θ=45° 时 AABB 面积是真实面积的 3.5×，这意味着:
1. 每辆旋转的 Truck 产生 3.5× 的多余正样本 cell
2. 这些多余 cell 的回归目标是噪声 (指向远离自身的位置)
3. 训练时这些噪声梯度与有效梯度竞争
4. 评估时这些 cell 产生额外 FP

修复优先级: CENTER/AROUND 标记 > 深度权重调制 > 修复 AABB (使用旋转投影而非 AABB)
