## Part 4: Plan D 结果的深度解读

### 4.1 truck_P = 0.202: 这是真正的突破吗？

**表面上**: truck precision 首次突破 0.20 目标。
**实质上**: 这个数字被 BUG-12 (评估 slot 排序不一致) 污染，但因为所有 Plan 都被同样的 bug 影响，相对比较仍然有效。

但更值得注意的是:
- truck_R = 0.148 (极低! 远低于 0.70 目标)
- truck_P = 0.202 意味着模型**非常保守地预测 truck** — 它很少预测 truck，但预测时比较准

这不是性能提升，而是**类别策略转移**: 模型从"大量猜测 truck"变为"很少猜测 truck"。

### 4.2 precision 和 recall 的反向运动

| 类别 | R 变化 | P 变化 | 解读 |
|------|--------|--------|------|
| car | +4.2% | -34% | 过度预测 car |
| truck | -16% | +248% | 欠预测 truck |
| bus | +90% | -29% | 过度预测 bus |

**模式**: 当模型开始更多预测某类时，recall 上升但 precision 下降（car, bus）。当模型减少预测某类时，recall 下降但 precision 上升（truck）。

**这说明模型的分类能力仍然很差。** 它没有学会"这里是 truck 还是 car"，只是在不同 Plan 中改变了预测阈值。

### 4.3 bg_FA = 0.207: Plan D 比 Plan C @500 差

Plan C @500 bg_FA = 0.156 (all-time best)
Plan D @500 bg_FA = 0.207 (+33%)

**原因分析**:
1. `resume=False` (BUG-10) 导致优化器冷启动，bg_fix 的效果被削弱
2. reg_loss_weight 从 2.0 降到 1.0 → 分类梯度增加，但这并不直接帮助背景识别
3. 从 Plan C iter_1000 恢复，但 iter_1000 的 bg_FA=0.171，已经比 @500 的 0.156 差

### 4.4 空间精度严重退化

offset_cy: 0.098 → **0.146** (+49%) — 已经超过 0.10 目标!
offset_cx: 0.064 → 0.089 — 接近 0.05 目标

**这验证了 1801 报告的预测**: reg_loss_weight=1.0 导致空间精度退化。但退化幅度超出预期 (offset_cy 增加了 49%)。

## Part 6: 判决

Plan D @500 的结果证实了我在 1801、1815、1825、1835 报告中的所有分析:

1. **梯度三重挤压** → car/truck 分类能力差 → precision 低 ✓
2. **优化器冷启动** → bg_FA 比 Plan C @500 差 ✓
3. **reg_loss_weight=1.0** → 空间精度退化 (offset_cy 已超标) ✓
4. **永久梯度裁剪** → 训练不稳定 → 类别策略偏移而非能力提升 ✓
5. **BUG-12 评估 slot 不一致** → precision 被人为压低 (未验证，需修复后重测)

### 给指挥家的建议

**立即修复 BUG-12 并重新评估所有 checkpoint。** 这是唯一可能改变局面的低成本操作。如果修复后 precision 翻倍（从 0.10 到 0.20），那么 Plan D 的真实表现可能已经接近目标。

如果修复后仍然不够，则需要架构性改变:
1. CENTER/AROUND 标记 (减少边缘 cell 噪声)
2. 修复 AABB 过估 (减少大物体的虚假 cell 覆盖)
3. 降低 clip_grad 敏感度 (提高 max_norm 或修复 loss 数值范围)

**停止继续在 loss weight 上打补丁。precision 问题的根因是评估 bug + 标签噪声，不是超参数。**
