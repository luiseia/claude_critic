## Q1: Center Cell ID 计算是否正确？ → **PASS (Minor Risk)**

### 代码路径
`generate_occ_flow_labels.py:517-527`:
```python
cx_img, cy_img = center_uv          # AABB 中心 = (min_u+max_u)/2, (min_v+max_v)/2
cc = int(np.clip(cx_img / fine_cell_w, 0, GRID_FINE_W - 1))
cr = int(np.clip(cy_img / fine_cell_h, 0, GRID_FINE_H - 1))
info['center_cell_id'] = cr * GRID_FINE_W + cc
```

### 判决
- `cr * GRID_FINE_W + cc`: 行优先线性化，与 `_compute_valid_grid_ids` 中的 `r * GRID_FINE_W + c` (line 285) 一致。**正确。**
- AABB 中心 vs 真实投影中心: AABB 中心 ≈ 8 个角点投影的 min/max 中点，不是 3D 框中心的精确投影。但对于近似正面的物体，误差 < 1 cell。**可接受。**
- 跨 grid 边界: `np.clip` 确保 cc, cr 在 [0, 19] 范围内。如果几何中心在图像外（投影出界），clip 会将 center_cell_id 推到边缘 cell。**边缘情况: 如果物体整体出界，`keep_front` 为 False，center_cell_id 不会被使用。安全。**
- **Minor Risk**: 对于旋转角极大的物体，AABB 中心可能偏离真实中心 1-2 cells。影响: center_weight 可能被分配给了错误的 cell。概率低、影响小。

## Q2: 权重乘法器应用位置是否正确？ → **PASS — 关键结论: 不会减少 Truck 总梯度**

### 代码路径
`git_occ_head.py:502-517`:
```python
for cid, gts in cell_to_gts.items():
    for i, gt in enumerate(gts[:NUM_SLOTS]):
        final_w = base_g_weights[gt['g_idx']] * energy_scale * pos_multiplier
        g_center = gt_center_cells.get(gt['g_idx'], -1)
        if g_center >= 0 and (self.center_weight != 1.0 or self.around_weight != 1.0):
            ca_mult = self.center_weight if cid == g_center else self.around_weight
            final_w = final_w * ca_mult
        tokens_weights[cid, base_idx : base_idx + 2] = final_w   # marker + cls
        tokens_weights[cid, base_idx + 2 : base_idx + 10] = final_w  # reg
```

### ca_mult 影响范围
ca_mult 应用到 `tokens_weights` 的所有 10 个 token 位置 (marker + cls + 8 个 reg)。这意味着 **marker loss、cls loss、reg loss 全部受影响**。

### 核心问题: Truck 覆盖 10 cells, 总梯度是否减少？

**表面计算** (指挥家的担忧):
- Pre-C/A: 10 cells × 1.0 = 总权重 10.0
- With C/A: 1 × 2.0 + 9 × 0.5 = 总权重 6.5
- 看似减少 35%!

**但这个计算是错误的。** 必须考虑 per_class_balance 的归一化效果。

### 关键数学推导

per_class_balance 的 marker loss 计算 (CE mode, line 906-919):
```python
loss_truck = Σ(CE_i × w_i × punish) / Σ(w_i)   # i ∈ truck cells
```

**w_i 同时出现在分子和分母中！** 这是一个加权平均，不是加权求和。

Pre-C/A (10 cells, 均匀权重):
```
loss_truck = Σ(CE_i × 1.0 × punish) / (10 × 1.0) = avg(CE_i) × punish
```

With C/A (center=2.0, around=0.5):
```
loss_truck = (2.0×CE_center + 0.5×Σ CE_around) × punish / (2.0 + 9×0.5)
           = (2.0×CE_center + 0.5×Σ CE_around) × punish / 6.5
```

如果所有 CE_i 相等: `= (2.0 + 4.5) × CE × punish / 6.5 = CE × punish` — **与 Pre-C/A 完全相同!**

### 梯度层面验证

单个 cell 的梯度:
```
∂loss_truck/∂logit_i = w_i × punish × (∂CE_i/∂logit_i) / Σ(w_j)
```

- Center cell: gradient ∝ 2.0 / 6.5 = 0.308 (vs pre-C/A: 1.0/10.0 = 0.100) → **3.08× 增强**
- Around cell: gradient ∝ 0.5 / 6.5 = 0.077 (vs pre-C/A: 0.100) → **0.77× 削弱**

总梯度（全部 10 cells 对共享参数的贡献）:
```
total = 1 × 0.308 + 9 × 0.077 = 0.308 + 0.693 = 1.001
pre-C/A total = 10 × 0.100 = 1.000
```

**总梯度守恒！Center/Around 在 per_class_balance 下不改变总梯度量，只改变梯度方向（更偏向 center cell 的 features）。**

### 判决

**PASS。指挥家关于"总权重 6.5 < 10.0 导致 truck 总梯度减少"的担忧在 per_class_balance 模式下不成立。** per_class_balance 的加权平均归一化保证了总梯度守恒。

Center/Around 的实际效果是：让 truck 的 center cell（特征质量最高）获得 3× 的梯度增强，让边缘 cell（特征噪声大）获得 0.77× 的梯度削弱。**这是正向的信噪比优化。**

同样的分析适用于 regression loss（`_per_class_reg_loss` → `_calculate_expectation_l1_loss` 也使用了 `(loss × weights).sum() / weights.sum()` 的加权平均结构）。

## Q5: 边界条件 → **PASS (Low Risk)**

### 单 cell 物体 (如远处 car)
- center=该 cell, around=无
- `final_w = IBW × energy × pos_mult × 2.0`
- per_class_balance: `CE × 2.0 × punish / 2.0 = CE × punish`
- **与 pre-C/A (CE × 1.0 × punish / 1.0) 完全相同。无影响。**

### 大物体 (如近处 trailer, 20+ cells)
- center 1 × 2.0 + around 19 × 0.5 = 11.5
- per_class_balance 归一化: `Σ(CE_i × w_i × punish) / 11.5`
- **per-class 总 loss 不变（加权平均归一化）。梯度集中到 center cell。**

### IBW 叠加
IBW = 1/N_cells。C/A 是在 IBW 之后叠乘。
- 20-cell trailer: IBW = 1/20 = 0.05
- Center cell final_w = 0.05 × energy × pos_mult × 2.0
- Around cell final_w = 0.05 × energy × pos_mult × 0.5
- per_class_balance 归一化后: 总 loss 不变

**PASS: 对大/小物体都公平（per_class_balance 归一化保证）。**

## Q6: BUG-9 (100% 梯度裁剪) 交互 → **CONDITIONAL PASS**

### 分析

clip_grad(max_norm=0.5) 对梯度的处理:
```
g_clipped = g × (0.5 / ||g||)    when ||g|| > 0.5 (100% of the time)
```

- 裁剪保留梯度**方向**，压缩**幅度**
- C/A 改变的是各 cell 梯度的**权重分配**，这改变了总梯度的**方向**
- 裁剪后，C/A 带来的方向变化被保留

### 具体效果

Pre-C/A: 总梯度 g = Σ(w_i × ∂loss/∂θ_i)，其中 w_i 均匀 → g 的方向受所有 cell 等权影响

With C/A: g' = Σ(w'_i × ∂loss/∂θ_i)，center 权重增大 → g' 的方向更偏向 center cell 的特征梯度

裁剪后:
- Pre-C/A: g_clipped = 0.5 × g/||g||
- With C/A: g'_clipped = 0.5 × g'/||g'||

**g' 和 g 的方向不同 → g'_clipped 和 g_clipped 的方向不同。C/A 的方向调整在裁剪后仍然生效。**

### 但效果被削弱

裁剪使所有迭代的步长固定为 max_norm=0.5，与梯度大小无关。C/A 的权重调整只改变方向，不改变步长。所以:
- **有效**: 梯度方向偏向 center features ✓
- **无效**: 无法通过权重增大来增加学习速度 ✗ (被裁剪抹平)

### 判决

**CONDITIONAL PASS。** C/A 在 100% 裁剪下仍有方向调制效果，但幅度效果被完全抹平。这意味着 C/A 的收益上界被 BUG-9 压低了。C/A 能提供的是"学什么"的优化，无法提供"学多快"的加速。
